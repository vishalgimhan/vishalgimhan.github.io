## 1. What is Computer Science?

At its core, Computer Science is not just about programming or learning a specific language like Python or C. It is defined as:

* The Study of Information: How we represent it and how we process it.
* Computational Thinking: Applying ideas from computer science to solve problems of interest.
* Problem Solving: The process of taking an Input (the problem) and using an algorithm to produce an Output (the solution).

## 2. Data Representation: Binary and Bits

To solve problems with computers, we must first agree on how to represent information using electronic devices.

* Unary Notation (Base-1): Counting using single digits (like fingers or tally marks).
* Binary System (Base-2): Computers use "switches" (transistors) that have only two states: On (1) or Off (0).
* Bit: Short for Binary Digit. It is the smallest unit of information a computer understands (a 0 or a 1).
* Byte: A group of 8 bits. A single byte can represent 256 different values ($2^8$).
* Counting in Binary: Instead of the "tens" and "hundreds" columns used in Decimal (Base-10), Binary uses powers of two: the 1s, 2s, 4s, 8s, 16s, 32s, 64s, and 128s columns.

## 3. Representing Complex Information

Once you can represent numbers with bits, you can represent anything by creating a standardized "mapping."

* Text (ASCII & Unicode):

  * ASCII: A standard mapping where specific numbers represent English letters (e.g., $65 = A$).
  * Unicode: A modern superset of ASCII that uses more bits (up to 32) to represent every character in every language, including Emojis.
* Images (RGB):

  * Images are made of Pixels (dots).
  * Each pixel is represented by three numbers: the amount of Red, Green, and Blue light.
* Video: A sequence of many images (frames) shown rapidly (e.g., 30 frames per second) to create the illusion of motion.
* Sound: Represented by numbers indicating the pitch, volume, and duration of notes.

## 4. Algorithms and Efficiency

An Algorithm is a precise, step-by-step description of how to solve a problem.

* Linear Search: Searching one by one from start to finish (e.g., page-by-page in a phone book). If there are $n$ pages, it might take $n$ steps.
* Binary Search (Divide and Conquer): Jumping to the middle, deciding if the target is in the left or right half, and throwing away the other half.

  * This is much faster: A 1,000-page book takes only about 10 steps.
  * Mathematically, this efficiency is described as $log_2 n$.

## 5. Programming Fundamentals (Pseudocode)

Before writing formal code, programmers often use Pseudocode—English-like instructions that mimic the logic of a program. Key constructs include:

* Functions: Actions or verbs (e.g., "Pick up," "Open," "Say").
* Conditionals: Forks in the road based on a question (e.g., "If," "Else if," "Else").
* Boolean Expressions: Questions that result in a True or False (Yes/No) answer.
* Loops: Cycles that allow you to repeat actions (e.g., "Go back to step 3").

## 6. Modern AI and Neural Networks

David touches on why we don't just "hard-code" every answer for things like chatbots:

* Neural Networks: Inspired by human biology, these are software structures that process massive amounts of training data.
* Probabilistic Logic: Instead of following a strict "If-Then" rule for every sentence, AI uses statistics to predict the most likely correct response to an input.

## 7. Abstraction

A major theme in CS50 is Abstraction—the idea of hiding complex details behind a simpler interface.

* Scratch: A graphical language that lets you learn logic without worrying about semicolons or syntax.
* Low-level vs. High-level: Zeros and ones are "low-level." Languages like C or Python are "high-level" because they allow humans to write instructions that are closer to English.
